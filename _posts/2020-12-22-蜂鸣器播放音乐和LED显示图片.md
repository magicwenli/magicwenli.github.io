### 电路原理

由图[1.1](#fig:beep1)所示的蜂鸣器电路可知，当BEEP引脚输出为低电平时，三极管导通，蜂鸣器鸣响；而当BEEP引脚输出为高电平时，三极管截止，蜂鸣器停止鸣响。

#### 任务1的分析与设计

任务1要求参考示例程序，自己编写程序实现蜂鸣器一长（响0.5秒停0.5秒）二短（响0.25秒停0.25秒）鸣响。

只需将蜂鸣器初始化后, 按照要求在main函数中调整延时即可.

```c
#include <avr/io.h>        // I/O端口寄存器配置文件，必须包含
#include <util/delay.h>    // 延时函数调用文件
int main(void)            // GCC中main文件必须为返回int的函数，无参数
{
	PORTE = 0X08;      // PORTE输出高电平，使蜂鸣器不响
	DDRE = 0X08;      // 配置端口PE3为输出口
	while (1) {
		PORTE &= ~(1 << PE3);  //PE3置"0"，蜂鸣器响
		_delay_ms(500);    // 延时500毫秒
			PORTE |= (1 << PE3);  //使PE3输出高电平，蜂鸣器不响
		_delay_ms(500);
		PORTE &= ~(1 << PE3);  
		_delay_ms(250);    // 延时250毫秒
			PORTE |= (1 << PE3);  
		_delay_ms(250);
		PORTE &= ~(1 << PE3);  
		_delay_ms(250);    // 延时250毫秒
			PORTE |= (1 << PE3);  
		_delay_ms(250);
		}
}
```

#### 任务2的分析与设计

##### 设置计数器和中断比较值

实验中需要使用计数/定时器T/C1比较匹配中断方式, 首先要设置计数器

```c
    // 计数器初始化
    TCCR1B = 0x00;     //中断控制寄存器清零，停止T/Cl计数  
    OCR1AH = 0x00;     //设置计数器比较值, 若匹配产生中断
    OCR1AL = 0x1e;     
    TCCR1A = 0x00;     //T/C1普通端口模式
    TCCR1B = 0x02;     //启动定时器T/C1，8分频
    TIMSK |= 0x10;     //使能C/Tl比较匹配中断
```

时钟频率为8MHz，T／C1设置为8分频，使用比较匹配中断.
计数值的计算方法如下：

$$\text{计数值} = \text{定时时间} \times \frac{\text{晶振频率}}{\text{分频值}}$$

$$\text{定时时间} = \frac{1}{2\times \text{声音频率}}$$

结合以上公式, 对于频率为500Hz的Do音, 中断计数值为

$$\text{counter}=\frac{1}{2\times500}\times \frac{8000000}{8}=1000=\text{0x03e8}$$

所以只需要控制中断比较值, 即可控制蜂鸣器音调.

```c
// 设置中断比较值
OCR1AH = 03;
OCR1AL = e8;
```

##### 通过函数设置比较值

根据上述结论, 给出控制中断比较值的函数如下：

```c
void getCouter(int f)
    {
      float numf = 1.0 / (2*f) * 8000000 / 8;
      OCR1AH = (int)numf >> 8;     // 取计数值的高8位
      OCR1AL = (int)numf & 0xff;   // 取计数值的低8位
    }
```

设置时钟中断程序如下：

```c
SIGNAL(SIG_OUTPUT_COMPARE1A)
    {
      TCNT1H = 0x00; //T/C1计数值清零
      TCNT1L = 0x00;
      //对PE3取反
      if (PORTE & _BV(PE3))
        PORTE &= ~_BV(PE3); //输出低电平
      else
        PORTE |= (1 << PE3); //输出高电平
    }
```

调用getCounter函数时, 蜂鸣器就会发出指定的音调.

接下来通过setMusic()函数解析音乐

```c
void setMusic()
    {
    	if(length>77){   //播放完毕, 从头开始
    		length=0;
    	}
        int height=HappyBirthday[length+1];		// 八度
        int weight=HappyBirthday[length]-1;		// 音色
    	int delay=HappyBirthday[length+2]*120;	// 延时

        {
        case 1:
            getCouter(mid[weight]);		// 中音
            break;
        case 2:
            getCouter(high[weight]);	// 高音
            break;
        case 0:
            getCouter(low[weight]);		// 低音
            break;
        
        default:
            break;
        }

        length+=3;
    }
```

#### LCD显示原理

##### LCD英文字符与汉字显示原理

当用LCD显示英文字符或汉字时，需要将相应的字模信息写入LCD
的显示数据缓存（DDRAM）中。所谓"字模信息"是指待显示的字符或汉字在空间上的像素点分布信息。一般的，英文字符的字模信息用16\*8的像素矩阵来表示（如图5-2），而单个汉字的字模信息则用16\*16的像素矩阵来表示（如图5-3）。LCD每一个显示单元均与特定的显示数据缓存器（DDRAM）地址区域相对应，只要在指定的显示数据缓存器（DDRAM）的某个地址区域写入了完整的字模信息，则与该地址区域对应的LCD显示屏上便会显示该字符或汉字了。

![LCD英文字符显示原理](/assets/images/2020-12-22-蜂鸣器播放音乐和LED显示图片/lcd21.png)

![LCD汉字显示原理](/assets/images/2020-12-22-蜂鸣器播放音乐和LED显示图片/lcd22.png)

以上是字符和汉字显示的基本原理，对于LCM128645ZK这款LCD显示器，实际的编程和使用中却没必要搜集字符或汉字的字模信息。因为LCM128645ZK自带了包含所有英文字符和8192个汉字的字模信息，编程时只需给出需要显示的字符或汉字就可以了，LCM128645ZK会根据您给出的字符或汉字查找字库中对应的字模信息，并写入指定的DDRAM地址区域中，使用起来非常方便。

##### LCD图像显示原理

LCM128645ZK的图像显示分辨率为128\*64，每个像素信息用1个bit来表示，图像画面的显示效果就像一幅128\*64大小的单色位图。图像的像素信息存放在LCD的图像数据缓存器（GDRAM）中，只要向GDRAM中写入图像的像素信息，便可以在LCD上显示对应的图像了。

显示图像时，LCD屏幕上像素点的位置与图像数据缓存器（GDRAM）的地址存在一一对应的关系。

```c
void LCD12864PortInit()
    {
      //LCD数据端口设置
      PORTB = 0x00; //
      DDRB = 0xFF;  //配置端口PB全部为输出口

      PORTD = 0x00;
      DDRD |= (1 << RS) | (1 << RW) | (1 << E);
      _delay_ms(15);
    }

    void LCD12864Init()
    {
      LCD12864WriteCommand(0x01); //清屏
      _delay_ms(15);
      LCD12864WriteCommand(0x38); //显示模式
      _delay_ms(5);
      LCD12864WriteCommand(0x0f); //显示开关控制，开显示，光标显示，光标闪烁
      LCD12864WriteCommand(0x0c); //显示开关控制，开显示，光标不显示，光标不闪烁
      _delay_ms(5);
      LCD12864WriteCommand(0x06); //光标设置,读/写一字符后,地址指针+1,光标+1,整屏不移动
      _delay_ms(5);
    }
```

每次显示图像内容都需要初始化, 这样才完成对屏幕的刷新.
将要显示的内容按行存放在数组中, 显示过程为
$$\text{初始化} \rightarrow \text{显示内容} \rightarrow \text{将数组逻辑右移} \rightarrow\text{初始化} \rightarrow \text{显示内容} \cdots$$

```c
函数名称：SIGNAL(SIG_OUTPUT_COMPARE1A)
功能：定时器/计数器1比较匹配中断的中断服务程序
说明：重置计数初值，并对PE3端口取反，开/关声响
入口参数：无
返回值：无
*********************************************/
SIGNAL(SIG_OUTPUT_COMPARE1A)
{
  TCNT1H = 0x00; //T/C1计数值清零
  TCNT1L = 0x00;
  //对PE3取反，0.5秒改变一次; x ^= 1;
  if (PORTE & _BV(PE3))
    PORTE &= ~_BV(PE3); //输出低电平
  else
    PORTE |= (1 << PE3); //输出高电平
}
//主程序
int main(void)
{
  length = 0;

  DDRE = _BV(PE3);   //初始化端口PE3为输出模式
  PORTE &= _BV(PE3); //输出高电平，蜂鸣器禁止发声
  TCCR1B = 0x00;     //中断控制寄存器清零，停止T/Cl计数
  OCR1AH = 0x00;     //设置8MHz、256分频、定时0.5s的比较值，
  OCR1AL = 0x1e;     //与TCNTl的计数值进行比较，若匹配产生中断
  TCCR1A = 0x00;     //T/C1普通端口模式
  TCCR1B = 0x02;     //启动定时器T/C1，256分频
  TIMSK |= 0x10;     //使能C/Tl比较匹配中断
  sei();             //允许全局中断
  while (1)
  {
    setMusic();
  } //等待中断的发生
}
```

```c
void wait_free()
{
	uint8_t retb;

	D_PORT = 0xFF;
	DDRB = 0x00;

	do
	{
		NOP();
		CLR_RS();
		NOP();
		SET_RW();
		NOP();
		SET_E();
		NOP();
		CLR_E();
		retb = PINB;
	} while ((retb & 0x80) != 0);
}

void write_comm(uint8_t cmd_dat, uint8_t cd)
{
	wait_free();
	DDRB = 0xFF;

	NOP();
	if (cd == DATA)
		SET_RS();
	else
		CLR_RS();

	NOP();
	CLR_RW();
	D_PORT = cmd_dat;
	NOP();
	SET_E();
	NOP();
	CLR_E();
}
```
